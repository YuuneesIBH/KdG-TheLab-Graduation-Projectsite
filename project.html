◈ PROJECT DNA

This project does not start from the idea of a simple launcher. It starts from the idea that a digital play environment can also be a physical, technical, and interactive installation.

One side delivers a complete cabinet flow with boot screens, CRT effects, local games, and gamepad controls. The other opens Guppy, a lab-oriented environment where the desktop app works directly with firmware, tools, scans, and external modules.

◈ FEATURE ARTICLE

The Lab Graduation: Arcade & Hacking Platform is a hybrid desktop project that deliberately brings together two very different worlds: the emotional pull of a classic retro arcade and the flexibility of an experimental hardware and security platform. Instead of behaving like a simple game launcher, the application treats play as part of a larger interactive installation.

The first layer is the arcade experience. From the moment the platform boots, the interface aims to feel like a real cabinet through fullscreen presentation, CRT-inspired effects, glitch animation, and gamepad-friendly navigation. Users can browse a local game library that already includes eleven built-in titles across Python, Java, and native runtimes, while the launcher keeps that diversity visually consistent inside one shared shell.

What makes the project stand out is Guppy, the second layer. Guppy transforms the same application into a terminal-style control environment that communicates over USB serial with a DIY ESP32-based board. From there, the system can expose functions such as IR commands, Wi-Fi scans, audit information, and access-point management while adapting the interface to the capabilities reported by the connected firmware.

A local hint layer extends the platform without taking focus away from the core experience. During gameplay, the system can generate short contextual hints through Ollama based on the selected title, genre, and difficulty. That keeps the feature practical and visible while still serving the cabinet flow instead of competing with it.

Taken together, the project demonstrates interdisciplinary building in a practical and visible way. Frontend design, desktop orchestration, embedded hardware, local tooling, and contextual guidance all meet inside one coherent cabinet experience. Security-oriented features remain intended for controlled, authorized demo or lab environments only.

◈ HINT LAYER & EXTENSIBILITY

The hint layer is deliberately supportive rather than dominant. During play, the platform can generate short contextual hints that make classic software more approachable without breaking the cabinet rhythm.

The architecture is also open enough to grow with the project. Additional guidance flows, synchronized sessions, or connected services can be added later without changing the core split between arcade and Guppy.

CURRENT USE
Local hints through Ollama based on game context and difficulty.
EXTENSIBLE SETUP
The same interface can later support richer guidance or connected services if the project grows beyond the local demo.
NEXT STEPS
Future multiplayer flows and synchronized demo moments fit naturally into the current structure.
SAFE CONTEXT
Security-oriented features remain limited to controlled and isolated learning environments.
◈ CONTEXT & VALUE
01
The security and network-oriented features are intended for controlled, authorized demo and lab environments. The project is explicitly not designed for use outside a permitted context.
02
What makes the project strong is the combination of disciplines: frontend, desktop orchestration, embedded firmware, physical interaction, and a local hint layer meet in one tangible installation.
03
The current version already demonstrates a working combination of launcher, local games, hardware bridge, and hint layer. It feels playful in form, but solid in structure.
04
In the longer-term vision, the platform can evolve toward a broader arcade ecosystem with synchronized sessions, richer demo tooling, and room for new cabinet experiences.
